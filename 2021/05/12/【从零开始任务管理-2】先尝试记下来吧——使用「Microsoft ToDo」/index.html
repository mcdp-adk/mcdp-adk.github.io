【从零开始任务管理#2】先尝试记下来吧——使用「Microsoft ToDo」

字数：750字

时长：2分钟
生活黑客

发布于：2021年5月12日
次浏览

先试着解放你的大脑

如果你从来没有用过任务管理软件，我建议你先着手试一下，只有真实地试过以后，你才会对任务管理有确实的认知。

而本文将简单介绍「Microsoft ToDo」（下称ToDo）及一些小技巧，便于你更快地建立初步认知。

基础功能

  1. 智能列表：「ToDo」内置的自动化筛选的列表。暂且提一句，笔者认为，自动化是任务管理的精髓所在，在「ToDo」中，你便能初步窥见自动化所带来的便利性，而在本文后半段，将提及笔者是如何利用智能列表提高效率的。
  2. 自定义列表：顾名思义，这是你的自定义区域。大概的逻辑是：建立「列表」以分类，去囊括你的「任务」；当然你也可以使用「组」去嵌套列表。
  3. 任务区域：你可以把「任务」作为最基础的单元，并对其做一些基础的操作。

使用技巧

  1. 对于「ToDo」，笔者的用法是直接使用列表进行分类，而对于一些杂乱的任务，直接放进「任务」列表中。
  2. 对于每一个具体的任务，笔者会根据实际需求，放入适合的智能列表中。如尚未到截止日期，但决定要做的，笔者会将其添加到「我的一天」中。通常情况下，笔者最经常使用的就是「我的一天」。
  3. 而对于任务的提前规划，笔者通常会查看「已计划日程」，按照时间顺序再一遍筛选如「我的一天」。
  4. 当然，将任务分类后直接在自定义列表查看也是可行的，笔者将其类似理解为「工作区」。
  5. 「ToDo」是可以进行简单的团队协作的，笔者一度将其作为安排学生工作的工具，安排、展示任务，在备注里写下任务要注意的事项。
  6. 在「ToDo」的设置中还有一些实用的小功能，例如当你将其固定在 Windows 的任务栏时，会显示角标，提醒你还有多少任务需要去完成。如果你将其固定在「开始」中，其磁贴会显示部分任务。

优点与缺点

笔者认为，「ToDo」最大的优点就是「轻」，你可以在几分钟内上手并摸索出适合你的任务管理方法；同时，「ToDo」是一个多平台的应用，无论是 Windows、Web 还是 Android 与 iOS 端，都有着不错的支持。

当然，缺点也十分明显，「ToDo」太「轻」了，不能满足用户的深度定制需求；同时，作为微软的应用，其同步速度被人诟病。

整体而言，「ToDo」作为一款入门的任务管理应用相当不错，但它作为微软的一款产品，用着总让人觉得膈应、不稳定，因此，笔者也就将其作为一款「轻」应用来使用了。
